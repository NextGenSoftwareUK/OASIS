# Simple NFT Minting Guide

## Overview

The NFT minting API has been simplified with smart defaults. You can now mint an NFT on Solana with minimal information!

## Simplest Example

**Just provide a title and you're done!**

```bash
curl -k -X POST "https://localhost:5004/api/nft/mint-nft" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "My First NFT"
  }'
```

## Smart Defaults

The API now automatically sets:

- **OnChainProvider**: `SolanaOASIS` (if not specified)
- **OffChainProvider**: `MongoDBOASIS` (if not specified)
- **NFTOffChainMetaType**: `OASIS` (if not specified)
- **NFTStandardType**: `SPL` for Solana, `ERC1155` for Ethereum (auto-detected)
- **Symbol**: `OASISNFT` (auto-generated by NFTManager if not provided)
- **NumberToMint**: `1` (default)
- **WaitTillNFTMinted**: `true` (default)

## Override Defaults (Optional)

You can still override any defaults if needed:

```json
{
  "title": "My NFT",
  "onChainProvider": "EthereumOASIS",  // Override default SolanaOASIS
  "offChainProvider": "IPFSOASIS",     // Override default MongoDBOASIS
  "nftStandardType": "ERC721",         // Override auto-detected type
  "symbol": "CUSTOM"
}
```

## Full Request Example

```bash
# 1. Authenticate
TOKEN=$(curl -k -s -X POST "https://localhost:5004/api/avatar/authenticate" \
  -H "Content-Type: application/json" \
  -d '{"username":"OASIS_ADMIN","password":"Uppermall1!"}' \
  | python3 -c "import sys, json; print(json.load(sys.stdin)['result']['result']['jwtToken'])")

# 2. Mint NFT (simplest form)
curl -k -X POST "https://localhost:5004/api/nft/mint-nft" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Test NFT"
  }'
```

## Notes

- `Symbol` is auto-generated as "OASISNFT" if not provided (handled by NFTManager)
- `JSONMetaDataURL` is only required if `nftOffChainMetaType` is set to `ExternalJsonURL`
- For `OASIS` meta type (the default), metadata is stored in MongoDBOASIS automatically
- The NFT will be minted to your authenticated avatar's wallet by default

## Error Handling

If you provide invalid values, you'll still get helpful error messages:

```json
{
  "isError": true,
  "message": "The OnChainProvider is not a valid OASIS NFT Provider..."
}
```

But with smart defaults, you rarely need to worry about these fields!
