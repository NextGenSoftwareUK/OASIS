# How the IP Pool Plan Connects to the OpenSERV / Hitchhikers Builders Fund Plans

**Purpose:** Map the IP Commons shared plan to the Builders Fund initiative (OpenSERV × Hitchhikers × OASIS) so both can be presented as one coherent plan for sign-on and fundraising.  
**Sources:** Hitchhikers folder (Builders Fund presentation, skeleton, David Bovill framing); IP Commons Shared Plan; Hitchhikers call 2026-02-21.

---

## 1. What the Builders Fund is — use the IP, contribute new IP to the pool

**Key docs:**  
- `Hitchhikers/OPENSERVE_BUILDERS_FUND_PRESENTATION_MARP.md` (deck)  
- `Hitchhikers/OPENSERVE_BUILDERS_FUND_PRESENTATION_SKELETON.md` (skeleton + speaker notes)  
- `Hitchhikers/DAVID_BOVILL_FRAMING_FEB_2026.md` (David B → Tim alignment)  
- `Docs/LFG/IP_Commons_Shared_Plan_Descriptive.md` (§6 Builders Fund)

**Primary purpose:** The Builders Fund is about **using the IP** (from the IP Pool) and **contributing new IP to the IP Pool**. In-person hackathons and events are **one way** to onboard and activate builders — not the definition of the fund.

**Use the IP:**
- Builders **license from the pool** and build with the open-core stack (holonic architecture, OpenSERV agent stack, Hitchhikers theme as overlay). They form companies, ship products, run ventures — all using the same IP the commons holds and licenses.
- Stack = OASIS holonic architecture + OpenSERV agent stack + (optional) Hitchhikers narrative. What they build lives in the Anarchive (holons); they can spin up entities with cap tables and tokenise agents (Pangea/Launchboard).

**Contribute new IP to the pool:**
- What builders create (agents, ventures, modules, data) can be **contributed or licensed into the IP Pool** on agreed terms (e.g. fair value, capped tithing back to the team). The pool grows with valuable new IP; creators get paid back the same way as OASIS and Hitchhikers contributors.
- So the Builders Fund is the **activation and growth layer** for the commons: use the IP, add new IP to the pool, tithing back to creators.

**Events (supporting, not central):**
- In-person or online events (e.g. hackathons, 42 locations, July 4, Burning Man) can support onboarding and community — but the fund’s identity is **use IP + contribute IP**, not “hackathon fund.”
- OpenSERV: agent stack, venture-builder narrative, optional event/prize infrastructure. Hitchhikers: theme, story, optional 42 locations and events. OASIS: holonic architecture, Anarchive, licensing from the pool.

## 2. Agent-First Equity (Pangea / Launchboard) — Cap Tables and Tokenised Agents

**Location:** `pangea-repo/docs/` — agent equity is already planned and partially integrated.

**Key docs:**
- `pangea-repo/docs/AGENT_FIRST_EQUITY_PLATFORM_OVERVIEW_2026.md` — product/GTM: tokenised capital for autonomous organisations; form entity, issue equity, deploy agents, run operations.
- `pangea-repo/docs/AGENT_FIRST_EQUITY_BUILD_PLAN_OASIS_A2A_PANGEA.md` — build plan: entity holon + lead agent, swarm, council, cap table, revenue attribution, distribution.
- `pangea-repo/docs/AGENT_FIRST_EQUITY_INTEGRATION_STATUS.md` — what’s integrated (Pangea cap table, OASIS tokenization, avatars) vs still needed (entity holon, lead agent, SERV, formation flow, revenue ledger).

**What it provides:**

- **Companies with cap tables:** Pangea Launchboard already has OCF-compatible cap table, organizations, stakeholders, share classes, tokenization (Web4 cap table NFT + Web3 share-class NFTs via OASIS). Entity formation flow (documents + sign + mint) is in scope.
- **Tokenised AI agents:** Agent-first design = one **entity holon** per org, **lead agent** (OASIS Avatar + agent holon), **swarm** of agents under that entity. Equity is minted **against** the entity (metadata: entity holon id, lead agent avatar id). So agents are not just “built” — they can sit inside a **legal entity with a cap table** and **tokenised equity**.
- **Revenue and distribution:** Build plan includes service ledger (agent work per entity), revenue attribution, entity treasury, and distribution to equity holders per cap table via OASIS wallets.

**Why include this in the Builders Fund:** So that teams don’t only build demos — they can **spin up real companies** (entity + cap table), **tokenise their AI agents** (equity against the entity), and **monetise** (revenue → treasury → distribution). The Builders Fund should make it **easy to create actual businesses and monetise work**; the agent-equity stack is the infrastructure for that.

**Link to IP Pool:** What builders create (agents, ventures, IP) can optionally be **contributed or licensed into the IP Pool** on agreed terms (e.g. tithing back to the team, or hybrid). So: Builders Fund → teams form entities, tokenise agents, generate revenue → a path exists to bring **builder-created IP** into the same commons that holds OASIS and Hitchhikers IP, with payback to creators.

## 4. One-page “structure + activation” story

- **Structure (IP Pool / Commons):** OASIS and Hitchhikers IP go into a neutral, not-for-profit commons. Tithing pays back creators; licensing is clear for governments and enterprises. Next Gen World (and others) **license** from the pool. OpenSERV, Hitchhikers, LFG sign the shared plan and get a clear answer on ownership and trust.
- **Activation (Builders Fund):** OpenSERV’s $1M builders fund + Hitchhikers’ 42 locations and theme + OASIS holonic architecture. The Builders Fund is **use the IP + contribute new IP to the pool**. Builders **license from the pool** and build with the open-core stack (OASIS holonic, OpenSERV agent stack, Hitchhikers overlay); what they create lives in the Anarchive (holons). They can **contribute** builder-created IP (agents, ventures, modules) **into the IP Pool** on agreed terms (tithing back to creators). **Agent-equity stack (Pangea/Launchboard)** lets teams spin up companies with cap tables and tokenise agents so they create **actual businesses** and monetise. In-person hackathons and events (e.g. 42 locations, July 4, Burning Man) are **one way** to onboard and activate — not the centre of the fund.
- **Fundraising:** We fundraise **collectively** for (1) the **commons** (stewardship, legal, tithing setup) and (2) the **Builders Fund** (use IP, contribute IP, optional events/prizes). Same signatories; same story: protect the IP, grow it with new contributions, activate use.
- **Decision point:** We take **one** plan — “IP pool + Builders Fund + shared roadmap — this is how we’re doing OASIS and Hitchhikers” — to OpenSERV (and LFG) and ask: **Are you in?** That locks commitment and funding for both structure and activation.
