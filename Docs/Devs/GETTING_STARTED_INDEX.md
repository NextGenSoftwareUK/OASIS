# Getting Started — Index by platform

## 🌐 Introduction to the WEB4 OASIS API

The **OASIS** (Open Advanced Sensory Immersion System) API is the core infrastructure powering Our World and manages a central profile/avatar/karma system. Satellite applications and games integrate with it to enable karma accumulation through good deeds and self-help activities.

The API provides access to authentication, karma management, avatars, data services, SEEDS, and wallet/key management. It bridges **WEB2 and WEB3** technologies across Holochain, IPFS, MongoDB, SQLite, Neo4j, Solana, Ethereum, EOS, ThreeFold and more — with additional networks planned.

As a **universal abstraction layer**, OASIS eliminates walled gardens and silos by connecting diverse protocols including Gab, Mastodon, SOLID, Fediverse, and ActivityPub. The WEB4 implementation operates as a fully distributed, decentralised, fault-tolerant, load-balanced system across the entire internet.

**Benefits of building on WEB4 OASIS:**
- **Simplified integration** — work with WEB2 and WEB3 without learning new tech stacks
- **Cross-blockchain compatibility** — the API handles interoperability between blockchains automatically
- **Future-proof architecture** — new technologies receive automatic integration and upgrades
- **WEB5 ready** — full support for the upcoming WEB5 STAR ODK ecosystem

The multilayer architecture features a REST API as the primary presentation layer (with gRPC and GraphQL endpoints planned). The WEB5 STAR ODK CLI provides a low-code metaverse generator on top.
