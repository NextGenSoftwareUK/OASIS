### Dev Environment Setup

Before anything else, complete the [OASIS Dev Environment Setup Guide](./DEVELOPMENT_ENVIRONMENT_SETUP.md) to prepare your local machine.

### Gitflow Workflow

The project follows a **Gitflow workflow**:

1. Create a new branch from the `development` branch for each task
2. Complete your work on your branch
3. Merge `development` into your branch **before** raising a Pull Request to prevent conflicts
4. Raise a Pull Request for review
5. After approval and testing in `development`, code moves to `master`

> **Important:** Maintain a balanced commit frequency â€” neither too many nor too few commits per task.

### Daily Dev Procedure

1. **Daily commits** â€” commit to your branch and pull/merge updates from `development` to stay current and avoid conflicts
2. **Task tracking** â€” keep tasks updated in [height.app](https://height.app)
3. **Daily standups** â€” post to your team's Discord channel in this format:

   > *"Yesterday I worked on X, today I am working on Y, no impediments"* (list any impediments if there are some)

This implements **SCRUM Agile methodology** to improve team coordination, transparency, and efficiency. Standups ensure work visibility, prevent duplication, facilitate peer support, and enhance overall communication.

**Team Discord channels:** `OASIS Web UI Dev`, `Our World Dev`, `One World Dev`, or `OASIS Backend Services Dev` on the Our World Tribe Server.
